5. NOTAS ADICIONALES PARA LA DISTRIBUCIÓN
========================================================================

  macOS:
  - Los .spec actuales NO tienen firma de código (codesign_identity=None)
  - Para distribución en macOS se necesita: Apple Developer ID,
    firma del binario, y notarización con Apple
  - Sin firma, macOS mostrará "app de desarrollador no identificado"
  - argv_emulation=True permite abrir archivos desde Finder

  Windows:
  - Antivirus puede dar falso positivo con binarios PyInstaller
  - Firmar el .exe con un certificado de code signing lo evita
  - Icono configurado: build/icons/waxcheck.ico

  Linux:
  - No hay .spec de build para Linux (habría que crear uno)
  - Alternativa: distribuir como AppImage o Flatpak
  - Dependencias de sistema: Tcl/Tk 8.6, portaudio, libsndfile

  Común:
  - Iconos configurados: .icns (macOS) y .ico (Windows) en build/icons/
  - datas[] incluye src/assets, customtkinter y tkinterdnd2
  - Los hidden imports están bien configurados para las 3 plataformas


6. PROBLEMAS DETECTADOS EN REVIEW (2026-03-19)
========================================================================

  6.1 PNG fallback incompleto en icon_utils.py (PRIORIDAD MEDIA)
  ─────────────────────────────────────────────────────────────────

  Cuando cairosvg no está disponible (típico en Windows), icon_utils.py
  intenta cargar PNGs como fallback. Pero los SVGs usan nombres V2/V3
  (ej: clean-iconV2.svg, spectrum-iconV2.svg) y no existen PNGs con
  esos nombres. Resultado: iconos transparentes/invisibles.

  Archivos afectados:
    - src/utils/icon_utils.py (líneas 54-61, lógica de fallback)
    - src/assets/ (faltan PNGs para los SVGs V2/V3)

  Fix: Exportar PNGs de cada SVG V2/V3 a src/assets/ con nombres
  coincidentes (ej: clean-iconV2.png, spectrum-iconV2.png, etc.)
  O bien incluir cairosvg + libcairo como dependencia obligatoria
  en el build de Windows.

  6.2 NSMicrophoneUsageDescription innecesario (PRIORIDAD BAJA)
  ─────────────────────────────────────────────────────────────────

  El .spec de macOS (build/waxcheck_macos.spec) incluye
  NSMicrophoneUsageDescription en info_plist. AudioQual no usa
  micrófono — solo reproduce audio. No causa problemas funcionales
  pero podría mostrar un prompt confuso al usuario.

  Fix: Eliminar la línea NSMicrophoneUsageDescription del info_plist
  en build/waxcheck_macos.spec.

  6.3 Constante huérfana OVERLAY_SPINNER_SPEED_MS (PRIORIDAD BAJA)
  ─────────────────────────────────────────────────────────────────

  analysis_overlay.py dejó de importar OVERLAY_SPINNER_SPEED_MS
  (el spinner ahora usa 16ms fijo para ~60fps). La constante
  probablemente sigue definida en src/utils/constants.py como
  código muerto.

  Fix: Eliminar OVERLAY_SPINNER_SPEED_MS de constants.py si no se
  usa en ningún otro sitio.
